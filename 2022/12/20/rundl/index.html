第一次跑通DL代码

配置环境 anaconda+pycharm
教程类
发布日期:   2022-12-20

前言

论文里的开源代码是不是想跑通？网上配置环境一大堆不知哪个可行？由于没有人带，走了很多弯路，踩了很多坑终于跑通了代码，在此记录一下！

  • 首先明确一点是在本地跑代码，要是在服务器上面就不需要这些软件啦，直接在服务器上配置就可。

  • 对电脑要求：要是英伟达显卡

  • 使用anaconda不需要再单独去python官网下载python

一、下载anaconda

  • 为什么安装anaconda：

Anaconda对于python初学者而言及其友好，相比单独安装python主程序，选择Anaconda可以帮助省去很多麻烦，Anaconda里添加了许多常用的功能包，如果单独安装python，这些功能包则需要一条一条自行安装，在Anaconda中则不需要考虑这些

  • 安装注意

安装的时候选择安装用户的时候一定选择Just me，之前看教程选择了all user会出现这个问题就是后续创建环境的时候，明明是安装在了D盘，按道理创建的环境应该在envs文件夹下，但是却在C盘，又百度了很多方法都不管用，最后知道安装时选择了all user怎么改都没有用，最后重新卸载安装选择just me正常。假如你的电脑有好几个 Users ，才需要考虑这个问题。其实我们电脑一般就一个 User，就我们一个人使用，所以直接选择just me。如果你的C盘足够大，这怎么选都无所谓呀

参考教程：http://t.csdn.cn/gqctL

四、在Pycharm中使用anaconda已创建好的环境

此时可以下载好一个开源的深度学习代码，右键用pycharm打开。

然后按照下面步骤进行。

选择add interpreter

选择conda environment，第一次找到你安装路径下面的conda.exe

事先创建好了的环境，应该是在conda安装目录下面envs文件夹里面，然后选择use existing environment，从下面选择你创建好的环境地址即可。或者事先没有创建好环境，选择create new environment 。也就是选择了一个python版本，默认地址就行。最后点击下面的ok就行了。

这个conda的作用是你每建立一个项目，都可设置一个新的环境，这样项目之间就不会发生环境冲突的问题了

自此所有环境配置完成，下面就可运行项目啦，点击pycharm上面的绿色三角按钮，下面肯定会报错，因为你还没有安装使用到的包，此时就可根据报错去安装所有的包啦（一般就是pip install 包名字），还有注意一下数据集的路径，有没有数据集文件在数据集的文件夹里面。

五、安装pytorch

因为我第一个运行的项目就是使用的pytorch框架，所以还要安装一下torch。

在我直接使用pip install torch安装torch时安装不成功。有的电脑就直接可以这样安装好（可能使用的是CPU）。我就查找了GPU方法，使用了下面这条指令即可

pip install torch===1.2.0 torchvision===0.4.0 -f https://download.pytorch.org/whl/torch_stable.html

因为我这个使用的torch版本是1.2的，对应的torchvision是0.4的。下面给出了torch和torchvision的版本对应关系。

参考链接：http://t.csdn.cn/fEKb4
